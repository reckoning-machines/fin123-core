fin123

Spreadsheet models that behave like software.
Build models in a spreadsheet. Run experiments like a program. Extend them with governed code.

Build

Author financial models in a familiar spreadsheet grid. Formulas, parameters, named ranges, multi-sheet workbooks. Canvas-rendered, keyboard-first, runs locally.

Everything you write is deterministic. Same inputs, same outputs, every time.

Operate

Terminal Mode turns your model into an executable program.

commitPersist state and execute a deterministic build. Returns a run_id.
scenarioLabel a committed run. Compare scenarios side by side.
sweepRun the model across one parameter. Each row is a real execution.
gridRun the model across two parameters. Matrix output.

A sweep runs the same model multiple times with different assumptions and collects the results. Each row is a real execution, not a recalculated cell.

sweep revenue_growth 0.04 0.06 0.08 0.10 grid revenue_growth 0.04 0.08 vs wacc 0.08 0.10 --output value_per_share

Extend

AI generates add-ins as reviewable Python code. Every artifact follows an explicit lifecycle:

# generate ai draft addin "calculate compound annual growth rate" # review and validate draft show draft_0001 validate draft draft_0001 # apply and build apply draft draft_0001 commit

Drafts are immutable. Revisions create new drafts. Nothing is applied silently. The user controls commit, build, and verify.

Architecture

Polars-backed scalar DAG and table evaluation. Lark LALR(1) formula parser. FastAPI local server. Canvas-rendered grid. Immutable runs with SHA-256 integrity. Plugin system with policy validation.

No database. No account. No network calls during builds.

Open source. Apache-2.0.
